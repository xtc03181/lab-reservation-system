# 开发计划

## 第 1 阶段：需求分析

- 明确学生、教师审核员、管理员三类角色
- 梳理实验室预约、设备借用、审核、公告维护等核心流程
- 完成用例图、业务流程图和功能模块划分

## 第 2 阶段：概要设计

- 确定前后端分离架构
- 设计数据库表结构
- 设计接口返回格式和异常处理方式
- 设计前端页面结构和菜单

## 第 3 阶段：详细设计与编码

- 后端完成实体类、Mapper、Service、Controller
- 前端完成登录、布局、数据表格和表单弹窗
- 完成预约审核和设备借用主流程

## 第 4 阶段：测试与完善

- 测试登录、实验室维护、设备维护、预约提交、审核等流程
- 补充分页查询、权限拦截、预约冲突校验、统计图表等功能
- 整理实习报告、数据库脚本和项目截图
