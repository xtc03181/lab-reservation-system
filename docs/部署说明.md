# 高校实验室预约申请与设备管理系统部署说明

本文档说明本项目在本地开发环境中的安装、配置、启动和常见问题处理方法。

## 一、环境要求

建议使用以下环境：

- JDK 17 或以上版本
- Maven 3.8 或以上版本
- Node.js 18 或以上版本
- MySQL 8.0 或兼容版本
- Redis 6.0 或以上版本
- IntelliJ IDEA
- Navicat Premium Lite 或其他 MySQL 管理工具

本项目采用前后端分离结构：

- 后端目录：`backend`
- 前端目录：`frontend`
- 数据库和文档目录：`docs`

## 二、数据库部署

### 1. 创建数据库

使用 Navicat 连接 MySQL 后，先新建数据库：

```sql
CREATE DATABASE lab_reservation DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

也可以直接运行 `docs/schema.sql`，脚本开头已经包含自动创建数据库语句。

### 2. 执行初始化脚本

在 Navicat 中打开查询窗口，执行：

```text
docs/schema.sql
```

脚本会创建以下核心表：

- `sys_user`：用户表
- `lab`：实验室表
- `equipment`：设备表
- `lab_reservation`：实验室预约表
- `equipment_borrow`：设备借用表
- `notice`：公告表
- `password_reset_code`：密码重置验证码表
- `operation_log`：操作日志表

初始化账号：

| 角色 | 账号 | 默认密码 |
| --- | --- | --- |
| 管理员 | `admin` | `Abc12345!` |
| 教师审核员 | `teacher` | `Abc12345!` |
| 学生 | `student` | `Abc12345!` |

如果数据库之前已经建好，不想清空原有数据，可以只执行增量脚本：

```text
docs/migration-20260626-enhancements.sql
```

该脚本会补充消息中心表和预约规则配置表，不会删除已有业务数据。

如果系统已经有实验室表，但还没有“开放星期/开放时间”字段，再执行：

```text
docs/migration-20260626-lab-open-time.sql
```

该脚本会给 `lab` 表增加：

- `open_days`：开放星期，1-7 分别表示周一到周日
- `open_start_time`：每日开放开始时间
- `open_end_time`：每日开放结束时间

## 三、后端配置与启动

### 1. 修改数据库配置

后端配置文件：

```text
backend/src/main/resources/application.yml
```

核心配置如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lab_reservation?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
```

如果你的 MySQL 密码不是 `123456`，需要改成自己的数据库密码。

### 2. Redis 配置

Redis 用于缓存公告列表，默认配置为：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      database: 0
```

如果本地没有启动 Redis，系统仍然可以运行，公告列表会自动回退到 MySQL 查询。

如果启动了 Redis，公告列表会优先从 Redis 读取，提高查询速度。

### 3. 邮箱验证码配置

忘记密码功能支持邮箱验证码。开发环境可以不配置真实邮箱，不配置时验证码会打印到 IDEA 后端控制台。

如果需要真实发送邮件，需要在 IDEA 的运行配置中设置环境变量：

```text
MAIL_HOST=smtp.qq.com
MAIL_PORT=587
MAIL_USERNAME=你的QQ邮箱@qq.com
MAIL_PASSWORD=你的邮箱授权码
```

注意：`MAIL_PASSWORD` 填邮箱授权码，不是邮箱登录密码。

### 4. 启动后端

使用 IDEA 打开 `backend` 目录，找到：

```text
com.gltu.labreservation.LabReservationApplication
```

运行 `main` 方法。

启动成功后，默认后端地址为：

```text
http://localhost:8080/api
```

如果 8080 端口被占用，可以修改：

```yaml
server:
  port: 8080
```

## 四、前端配置与启动

### 1. 安装依赖

进入前端目录：

```bash
cd frontend
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

默认前端访问地址：

```text
http://localhost:5173
```

### 3. 后端接口地址

前端接口配置在：

```text
frontend/src/api/request.js
```

默认接口地址：

```javascript
baseURL: '/api'
```

开发环境由 `vite.config.js` 代理到 `http://localhost:8080`，Docker 环境由 Nginx 代理到后端容器。

## 五、常用运行步骤

完整启动顺序建议如下：

1. 启动 MySQL。
2. 使用 Navicat 执行 `docs/schema.sql`。
3. 启动 Redis。
4. 启动后端 `LabReservationApplication`。
5. 进入 `frontend`，执行 `npm run dev`。
6. 浏览器访问 `http://localhost:5173`。
7. 使用 `admin / Abc12345!` 登录管理员账号。

## 六、生产构建

### 1. 后端打包

进入后端目录：

```bash
cd backend
mvn clean package -DskipTests
```

打包结果：

```text
backend/target/lab-reservation-backend-0.0.1-SNAPSHOT.jar
```

运行 jar：

```bash
java -jar target/lab-reservation-backend-0.0.1-SNAPSHOT.jar
```

### 2. 前端打包

进入前端目录：

```bash
cd frontend
npm run build
```

打包结果：

```text
frontend/dist
```

正式部署时可以将 `dist` 放到 Nginx 中。

## 七、Docker 一键部署

项目根目录已经提供：

- `docker-compose.yml`：统一启动 MySQL、Redis、后端、前端
- `backend/Dockerfile`：后端打包并运行 Spring Boot
- `frontend/Dockerfile`：前端打包并交给 Nginx 访问
- `frontend/nginx.conf`：支持前端刷新不 404，并把 `/api` 转发到后端

安装 Docker Desktop 后，在项目根目录执行：

```bash
docker compose up -d --build
```

首次启动时，MySQL 容器会自动执行：

```text
docs/schema.sql
```

启动完成后访问：

```text
http://localhost
```

默认账号：

```text
admin / Abc12345!
teacher / Abc12345!
student / Abc12345!
```

如果要停止所有容器：

```bash
docker compose down
```

如果想连同 Docker 里的数据库数据一起清空：

```bash
docker compose down -v
```

注意：`docker compose down -v` 会删除 Docker 中的 MySQL 数据卷，演示数据会被清空。

## 八、常见问题

### 1. 后端启动失败，提示端口 8080 被占用

原因：已有后端服务正在运行。

解决方法：

- 停止旧的后端进程。
- 或修改 `application.yml` 中的 `server.port`。

### 2. 数据库连接失败

检查：

- MySQL 是否启动。
- 数据库名是否为 `lab_reservation`。
- `application.yml` 中用户名和密码是否正确。
- Navicat 是否能正常连接同一个数据库。

### 3. 登录失败

检查：

- 是否执行过 `docs/schema.sql`。
- 默认账号密码是否正确。
- 用户状态是否为启用。

默认账号：

```text
admin / Abc12345!
teacher / Abc12345!
student / Abc12345!
```

### 4. 邮箱验证码收不到

可能原因：

- 没有配置 `MAIL_USERNAME` 和 `MAIL_PASSWORD`。
- `MAIL_PASSWORD` 填成了邮箱登录密码，而不是授权码。
- QQ 邮箱未开启 SMTP 服务。
- 网络或 SMTP 服务连接超时。

如果没有配置真实邮箱，验证码会打印在后端控制台。

### 5. Redis 没启动会不会影响系统

不会。

本项目中的 Redis 用于公告列表缓存。如果 Redis 无法连接，系统会自动回退到 MySQL 查询，业务功能仍然可用。

### 6. Docker 启动后访问哪个地址

Docker 部署后前端访问：

```text
http://localhost
```

后端接口地址仍然是：

```text
http://localhost:8080/api
```

前端页面中请求 `/api` 会由 Nginx 自动转发到后端。

### 7. 实验室开放时间如何生效

管理员进入“实验室信息”，给每个实验室设置开放星期和每日开放时间。

学生提交预约时，后端会校验：

- 预约不能跨天
- 预约日期必须属于实验室开放星期
- 开始时间和结束时间必须在实验室开放时段内

## 九、部署检查清单

正式演示前建议检查：

- MySQL 已启动。
- 数据库脚本已执行。
- 后端启动无报错。
- 前端能访问登录页。
- 管理员、教师、学生三个账号都能登录。
- 实验室预约、设备借用、审核、公告、导出、日历视图正常。
- 实验室开放星期和开放时间校验正常。
- Redis 启动后公告列表能正常访问。
- 邮箱验证码功能根据实际环境选择真实发送或控制台打印。
